自动日报/周报与复盘

了解 Project Mirror

Project Mirror 是一个 Windows 托盘常驻的本地回顾工具：自动收集你电脑上的工作“痕迹”（窗口、代码变更、浏览历史），整理成可追溯的工作片段，并生成日报/周报与技能变化。

你能得到什么

  • 每天/每周的回顾：每条结论都能点开看到证据覆盖与明细。
  • 技能树与能力变化：增长来源可追溯到贡献的工作片段。
  • 离线也能用：不配 AI Key 也能生成规则版摘要与回顾。
  • 遇到“空白页/没数据”也不需要找作者：状态页会告诉你缺口在哪一层，并给出下一步怎么修。

隐私与本地优先

  • 数据默认写入本地 SQLite（默认 ./data/mirror.db），不做云同步/账号体系。
  • 本地 HTTP 仅监听 127.0.0.1，端口随机分配。
  • 启用 AI 时仅发送脱敏后的最小必要文本，用于生成摘要与建议。

它以什么形式运行（v0.2）

  • mirror.exe（Windows 10/11 托盘常驻）
  • 内置本地 HTTP Server（随机端口，仅监听 127.0.0.1）
  • 内置 Web UI（React/Vite SPA，优先加载同目录 frontend/dist/，否则回退内嵌资源）
  • 端口发现：写入 ./data/http_base_url.txt

它怎么工作（简述）

采集 → 生成工作片段（切分/聚合/摘要）→ 汇总到技能与回顾 → 每条结论可点开证据链。

目前不包含什么

  • 云同步/账号体系/团队协作
  • macOS/Linux（当前阶段）
  • 复杂目标管理（TODO/OKR/习惯）

开发者：目录结构（Go / React）
  • cmd/mirror-agent/：Windows Agent 入口（托盘、采集器、HTTP API/UI）
  • internal/collector/：采集器（窗口/Diff/浏览历史）
  • internal/server/：本地 HTTP Server（JSON API + SSE + 静态 UI）

  • frontend/：前端源码（开发时通过 VITE_API_TARGET 指向本地 Agent）

口径说明

上层的技能/回顾不直接从 raw 统计，而是统一以会话为输入，保证“同一套数字、同一条证据链”。

出了问题怎么自助排查

当数据为空/不可信时，优先从状态页定位缺口（采集/切分/分析/配置），并按提示执行：补配 watch paths、重建某天会话、导出诊断包等。

推荐阅读

快速开始

Project Mirror 是一个 Windows 托盘常驻 Agent + 本地 Web UI。目标是让新用户在 10 分钟完成闭环：配置 watch paths → 看到当天会话 → 能点开证据。

1. 获取程序

推荐从 GitHub Releases 下载 Windows 版本；或从源码构建（需 Go 1.25.4，且仅支持 Windows 构建）。

go build -trimpath -ldflags "-H=windowsgui -s -w" -o .\mirror.exe .\cmd\mirror-agent\

说明：上述命令会构建为 GUI 子系统，运行时不会弹出前台控制台窗口。

2. 初始化

首次运行后，Project Mirror 会最小化到系统托盘。右键点击图标选择 "打开面板"。

  • 默认数据库：./data/mirror.db（相对路径以可执行文件目录为基准）
  • 本地 HTTP：只监听 127.0.0.1，端口为随机分配
  • 端口发现：启动后写入 ./data/http_base_url.txt

推荐“便携分发”：一个 mirror.exe + 同目录 config/ 与 data/。

3. 配置监听目录

为了让 Diff Collector 生效，你需要在设置页添加你的 Git 项目目录（watch paths）：

C:\Users\Dev\Projects\Go-Backend

可选：启用浏览器采集（自动探测 Chrome/Edge History 路径），并保持隐私脱敏默认开启。

本地接口（高级）

本地 HTTP 仅监听 127.0.0.1，端口随机分配（见 ./data/http_base_url.txt）。

状态与可观测性（P0）

  • GET /api/status：采集/管道/AI/最近错误的汇总快照
  • GET /api/events：SSE（数据变更/配置变更等事件）

维护动作

  • POST /api/maintenance/sessions/rebuild：按日期重建会话
  • POST /api/maintenance/sessions/enrich：按日期补全语义（启用 AI 时）
  • GET /api/diagnostics/export：导出脱敏诊断包（zip）

常见问题

需要联网吗？

不需要。未配置 AI Key 时，Project Mirror 会以规则口径生成会话摘要与回顾；配置 AI 后，仅在需要语义化摘要/建议时调用外部提供商。

为什么页面为空/没有数据？

  • 先打开 /status（或状态页入口）查看采集健康与最近错误。
  • Diff 采集需要配置 diff.watch_paths（或在设置/引导里选择项目目录）。
  • 浏览器采集依赖本地 History 文件路径；可在设置中查看自动探测结果并手动指定。

支持 macOS/Linux/手机吗？

当前阶段只支持 Windows 10/11，且不提供云同步/账号体系。

如何导出诊断信息？

在状态页使用“导出诊断包”，或访问 /api/diagnostics/export 下载脱敏后的 zip。

隐私策略

最后更新：2025年12月

Project Mirror 遵循 Local-first & Privacy by default：默认本地存储、默认脱敏、默认可离线。

数据存储

采集数据（窗口事件、Diff、浏览历史等）会写入本地 SQLite 数据库（默认 ./data/mirror.db）。本地 HTTP 仅监听 127.0.0.1，不会暴露到局域网/公网。

默认脱敏

默认开启脱敏：URL 会清空 query/fragment 并保留域名与“存在路径”的事实；窗口标题/浏览标题会按规则替换敏感片段。你可以在配置中调整 privacy.patterns。

AI 处理

不配置任何 Key 时，不会调用外部 AI。启用 AI 后，仅在需要生成语义摘要/建议时发送经脱敏的最小必要文本到你配置的提供商（例如 DeepSeek / SiliconFlow）。

设置与隐私

配置文件为 YAML：默认路径 ./config/config.yaml（首次启动会自动生成）。可以参考模板 ./config/config.yaml.example。

你大概率只需要改这几项

diff:
  enabled: true
  watch_paths:
    - "C:\\Users\\Dev\\Projects\\Go-Backend"

browser:
  enabled: true
  history_path: "" # 为空则自动探测

storage:
  db_path: "./data/mirror.db"

privacy:
  enabled: true

AI Key 与环境变量

建议通过环境变量注入 Key（避免写入磁盘/提交仓库），例如 DEEPSEEK_API_KEY、SILICONFLOW_API_KEY。
